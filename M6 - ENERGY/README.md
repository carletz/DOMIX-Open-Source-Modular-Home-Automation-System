## Overview
The DOMIX M6 module measures electrical consumption using clamp meters. It also measures the current mains voltage to accurately calculate the power used at the measurement points.

## System Architecture
The architecture, with three PCBs connected together, is the same as the previous modules. The base has the main task of implementing the measurement logic, the vertical part is responsible for communication with the main module, and the upper part is responsible for the display.

Voltage measurement is performed via the ZMPT101B single-phase voltage transformer (AC voltage max. 250 volts) present at the input. The output signal is sinusoidal, conditioned by an operational amplifier (LM358), and centred around a DC offset of approximately 1.25V (VREF/2 generated by a CJ431 reference).

Current measurement is carried out through a conditioning circuit with an operational amplifier (LMV321) which generates a 1V DC bias, followed by an analog multiplexer CD74HC4067SM96 that probes all 10 available channels. Channel selection is handled by a PCA9554APW,118 I/O expander over I2C.

### ADC Architecture — ATtiny1616

Starting from hardware revision 1.1, the ADS1115 ADC previously used for both voltage and current measurement has been replaced by an **ATtiny1616 microcontroller** (QFN-20-EP, 3×3 mm). This change was necessary because the ADS1115 uses a sigma-delta converter that internally averages the input signal, making it unable to follow a 50 Hz AC waveform — it always returned the DC offset instead of the RMS value.

The ATtiny1616 uses a SAR ADC capable of over 75,000 samples per second, providing approximately 1,500 samples per 50 Hz cycle. It computes the RMS value internally using a two-pass algorithm (DC offset removal followed by RMS calculation) and exposes the results to the ESP32 via I2C, emulating the same address as the original ADS1115 (0x48).

#### ATtiny1616 Pin Assignment

| QFN pin | Port | Function | Connected to |
|---------|------|----------|-------------|
| 1  | PA2 | SCL (TWI0 default) | IIC_SCL |
| 9  | PB5 | AIN8 / ADC | READ (ZMPT101B voltage signal) |
| 10 | PB4 | AIN9 / ADC | ADCCOM (CT clamp current via MUX) |
| 19 | PA0 | UPDI | Programming pad (3-pin: VCC / GND / PA0) |
| 20 | PA1 | SDA (TWI0 default) | IIC_SDA |
| EP | —   | Thermal pad | GND plane (mandatory) |

All other pins are left unconnected (NC).

#### I2C Register Map

| Register | R/W | Description |
|----------|-----|-------------|
| 0x00 | R | Voltage RMS (float, 4 bytes big-endian) |
| 0x01 | R | Current RMS (float, 4 bytes big-endian) |
| 0x02 | W | Trigger new current acquisition (write 0x01) |
| 0x03 | R | Status: 0x00 = busy, 0x01 = ready |

#### Current Acquisition Protocol

Because the CD74HC4067 multiplexer channel is selected by ESPHome before each reading, current acquisition must be explicitly triggered to avoid reading stale samples from a previous channel:

1. ESPHome selects the channel on CD74HC4067 via PCA9554
2. ESPHome writes 0x01 to register 0x02 (TRIGGER)
3. ATtiny1616 clears its buffer and acquires a fresh RMS measurement
4. ESPHome polls register 0x03 until STATUS = 0x01 (ready)
5. ESPHome reads register 0x01 (current RMS result)

Voltage acquisition is continuous and requires no trigger — register 0x00 always returns the latest computed value.

#### ADC Reference and Signal Range

The ATtiny1616 uses the internal 2.5V reference (INTERNAL2V5). Signal ranges at the ADC inputs are:

| Signal | DC offset | AC swing (peak) | Min | Max |
|--------|-----------|-----------------|-----|-----|
| READ (voltage) | 1.244V | ±0.707V | 0.537V | 1.951V |
| ADCCOM (current, CT 50A/1V) | 0.643V | ±0.141V (at 5A) | 0.502V | 0.784V |

Clipping of the current signal begins above approximately 36A on a single channel.

#### Firmware

The ATtiny1616 is programmed using Arduino IDE with the **megaTinyCore** board package. Clock is set to 10 MHz internal (safe maximum at 3.3V per datasheet). Calibration factors `CAL_VOLTAGE` and `CAL_CURRENT` are defined as compile-time constants and must be updated after initial measurement against a reference meter, then re-flashed via UPDI.

Programming interface: any USB-Serial adapter (CH340, CP2102) with a **470Ω resistor** between TX and the UPDI pad. No reset or boot button required.

Arduino IDE settings:
- Board: `ATtiny1616`
- Clock: `10 MHz Internal`
- Programmer: `SerialUPDI - SLOW: 57600 baud`

## PCA9554A I/O Expander — Address Selection

The PCA9554A controls the 4 address lines (S0–S3) of the CD74HC4067 multiplexer. Its I2C address is set via the DIP switch SW1 on the side of the board:

|Address | A2 | A1 | A0 |
|--------|----|----|-----|
| 0x38   | 0  | 0  | 0  |
| 0x39   | 0  | 0  | 1  |
| 0x3A   | 0  | 1  | 0  |
| 0x3B   | 0  | 1  | 1  |
| 0x3C   | 1  | 0  | 0  |
| 0x3D   | 1  | 0  | 1  |
| 0x3E   | 1  | 1  | 0  |
| 0x3F   | 1  | 1  | 1  |

The same table is reproduced as silkscreen on the back of the PCB.

## ATtiny1616 I2C Address Selection

The ATtiny1616 I2C address is defined in firmware (`Wire.begin(0x48)` by default). If address 0x48 conflicts with another device on the bus, change the `I2C_ADDRESS` define in the firmware and re-flash via UPDI.

## Power Supply Architecture
The module's control logic is powered by the IDC connector from the DOMIX M1 core module. The LM358 operational amplifier used for voltage measurement requires 5V, supplied from the same connector. The ATtiny1616 and all other control logic operate at 3.3V. The 1V DC bias for the current conditioning circuit is generated by the LMV321 operational amplifier configured as a voltage follower with a 22kΩ/10kΩ resistive divider from 3.3V.

## User Interface & Display Module
A user interface made with a display identical to the DOMIX M1 module is foreseen in the project. Its use should be carefully evaluated due to the limited number of available I2C addresses on the bus. Adding another I2C screen could cause address conflicts; each installation should be planned case by case.

## Mechanical Structure
Like all other DOMIX modules, the DOMIX M6 system uses a stacked PCB architecture: the lower PCB contains the core electronics, the vertical PCB provides mechanical and electrical interconnect, and the upper PCB houses the human interface. This structure minimises footprint, improves maintainability, and allows for future UI or expansion board development.
